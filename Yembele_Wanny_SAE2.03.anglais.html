SAE 2.03 - Setting up an IT solution for the enterprise

Design and Deployment of a Network Inventory Management Web Application

📋 General Presentation

The objective of this SAE was to design, develop and deploy a complete dynamic web application to monitor, track, and map the network equipments of an organization (such as an IUT department).

The project integrates a relational database architecture, a dynamic application server, and a multi-service containerized deployment via Docker.

🛠️ Architecture and Technologies

The infrastructure relies on a set of containerized services ensuring modularity and ease of deployment:

  • Application Framework: Python avec Flask pour la gestion des routes, des sessions de connexion et du rendu dynamique via le moteur de templates Jinja2.
  • Database: PostgreSQL 14, assurant la persistance des données.
  • ORM: Flask-SQLAlchemy pour interfacer le code Python avec la base PostgreSQL de manière sécurisée et typée.
  • Security: User password hashing via Werkzeug Security (PBKDF2/Scrypt).
  • Containerization (Docker & Compose):
    • web : Conteneur Flask hébergeant l'application Python.
    • db : Conteneur PostgreSQL hébergeant la base de données relationnelle.
    • pgadmin : Interface web d'administration de la base de données.

🗃️ Data Model

The relational database contains three main tables:

  1. Equipement : Stores device details (name, equipment type [Switch, Router, PC, Printer, Phone], brand, IP address, room location, and status [Active, Inactive]).
  2. Utilisateur : Manages credentials (username, email address, hashed password, and est_admin administrator role).
  3. Historique : Logs all actions performed on the devices (user ID, equipment ID, action [Add, Edit, Delete], and timestamp).

🚀 Key Features

1. Authentication and Access Control

The application offers a secure signup and login system. Critical modifications (Add, Edit, and Delete) are strictly restricted to admin users, while standard users can only view the inventory and the cartography.

2. Dynamic Dashboard (Home)

The home page displays real-time key statistics on the network status:

  • Total number of registered devices.
  • Ratio of active (Running) vs inactive (Stopped) devices.
  • Quantitative distribution of devices by type (Switches, Routers, PCs, Printers, Phones).

3. Filtered Inventory and Advanced Pagination

The inventory displays all devices in an interactive table. Users can filter by type, brand, room, or status. To optimize performance, the list uses SQLAlchemy pagination (10 items per page) while dynamically preserving active search filters across pages using query parameters.

4. Device Mapping (Buildings)

A visual grid layout organizes devices by buildings (H, I, J) and rooms (e.g., H0-05, H1-11, I0-02, J0-01, J0-04, J0-09) as well as a Storage room. Each room shows the count of devices and acts as a dynamic link to pre-filter the inventory for that room.

5. Audit Log / History

A paginated audit log allows admins to track interventions. Action timestamps are automatically recorded in Paris (France) timezone using the zoneinfo library.

🎨 Design and User Interface (UI/UX)

The interface has been completely redesigned to offer a premium "Tech Console" aesthetic:

  • Thème visuel : Style sombre (dark space blue) avec un effet de transparence "glassmorphism" (panneaux floutés avec bordures lumineuses subtiles).
  • Typographie : Utilisation de la police moderne Outfit.
  • Badges de Statut : Codes couleur harmonieux pour identifier immédiatement le statut (vert pour actif, rose pour inactif) et la salle.
  • Micro-animations : Transitions fluides et élévations des cartes au survol de la souris.
