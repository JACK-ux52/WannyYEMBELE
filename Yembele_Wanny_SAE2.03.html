SAE 2.03 - Mettre en place une solution informatique pour l'entreprise

Conception et Déploiement d'une Application Web de Gestion de Parc Réseau

📋 Présentation Générale

L'objectif de cette SAE était de concevoir, développer et déployer une application web dynamique complète permettant d'assurer la supervision, le suivi et la cartographie des équipements réseau d'une organisation (comme un département d'IUT).

Le projet intègre une architecture de base de données relationnelle, un serveur d'application dynamique et un déploiement conteneurisé multi-services via Docker.

🛠️ Architecture et Technologies

L'infrastructure repose sur un ensemble de services conteneurisés garantissant la modularité et la facilité de déploiement :

  • Framework Applicatif : Python avec Flask pour la gestion des routes, des sessions de connexion et du rendu dynamique via le moteur de templates Jinja2.
  • Base de Données : PostgreSQL 14, assurant la persistance des données.
  • ORM : Flask-SQLAlchemy pour interfacer le code Python avec la base PostgreSQL de manière sécurisée et typée.
  • Sécurité : Hachage des mots de passe utilisateurs via Werkzeug Security (PBKDF2/Scrypt).
  • Conteneurisation (Docker & Compose) :
    • web : Conteneur Flask hébergeant l'application Python.
    • db : Conteneur PostgreSQL hébergeant la base de données relationnelle.
    • pgadmin : Interface web d'administration de la base de données.

🗃️ Modèle de Données

La base de données relationnelle contient trois tables principales :

  1. Equipement : Stocke les caractéristiques des machines du parc (nom, type d'équipement [Switch, Routeur, PC, Imprimante, Téléphone], marque, adresse IP, localisation [salle], et statut [En marche, À l'arrêt]).
  2. Utilisateur : Gère les accès (nom d'utilisateur, adresse email, mot de passe haché et indicateur de rôle administrateur est_admin).
  3. Historique : Enregistre l'ensemble des actions effectuées sur le parc (ID de l'utilisateur, ID de l'équipement, action [Ajout, Modification, Suppression] et horodatage).

🚀 Fonctionnalités Clés

1. Authentification et Contrôle d'Accès

L'application propose un système d'inscription et de connexion sécurisé. Les actions critiques de modification du parc (Ajout, Modification et Suppression) sont strictement réservées aux utilisateurs possédant le rôle d'administrateur, tandis que les utilisateurs standard peuvent consulter l'inventaire et la cartographie.

2. Tableau de Bord Dynamique (Accueil)

La page d'accueil affiche en temps réel des statistiques clés sur l'état du réseau :

  • Nombre total d'équipements enregistrés.
  • Proportion d'équipements actifs (En marche) vs inactifs (À l'arrêt).
  • Répartition quantitative du matériel par type (Switchs, Routeurs, PC, Imprimantes, Téléphones).

3. Inventaire Filtré et Pagination Avancée

L'inventaire permet de visualiser l'ensemble du parc sous forme de tableau interactif. L'utilisateur peut filtrer les résultats par type, marque, salle ou statut. Pour optimiser les performances, l'affichage utilise la pagination SQLAlchemy (10 éléments par page) tout en **conservant dynamiquement les filtres actifs** d'une page à l'autre grâce aux paramètres de requête.

4. Cartographie du Parc (Bâtiments)

Une représentation visuelle sépare les équipements par bâtiments (H, I, J) et par salles (ex: H0-05, H1-11, I0-02, J0-01, J0-04, J0-09) ainsi qu'une zone de Réserve. Chaque salle affiche le nombre de machines présentes et sert de lien dynamique pour pré-filtrer l'inventaire sur cette salle.

5. Journal d'Audit / Historique

Un panneau d'historique paginé permet aux administrateurs de suivre les interventions sur le parc. Les dates et heures des actions sont automatiquement enregistrées selon le fuseau horaire de Paris (France) grâce à la bibliothèque zoneinfo.

🎨 Design et Interface Utilisateur (UI/UX)

L'interface a été entièrement révisée pour offrir une esthétique premium de type "Console Technique" :

  • Thème visuel : Style sombre (dark space blue) avec un effet de transparence "glassmorphism" (panneaux floutés avec bordures lumineuses subtiles).
  • Typographie : Utilisation de la police moderne Outfit.
  • Badges de Statut : Codes couleur harmonieux pour identifier immédiatement le statut (vert pour actif, rose pour inactif) et la salle.
  • Micro-animations : Transitions fluides et élévations des cartes au survol de la souris.
